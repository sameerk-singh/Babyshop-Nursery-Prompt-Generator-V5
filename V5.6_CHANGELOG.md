## Why "removed" things kept reappearing

I was patching the aesthetics pool only. The actual issues lived in the **walls** and **lighting** pools too. The cloud nightlight in Carry Cot and Swing was coming from a `Warm Nightlight Glow` entry in the lighting pool — totally independent of which aesthetic was chosen. The deep green walls were coming from `Saturated Matte Accent Wall` in the walls pool. Same for Color-Block Studio Plane wall and Arched Alcove Plaster wall. None of those were touched by V5.3-5.5 changes.

V5.6 cleans all three pools.

### D. New Feeding Pillow scale directive

```
FEEDING PILLOW SCALE — PROPORTIONAL
The feeding pillow rests on a rug, ottoman, or armchair seat at moderate
camera distance. It occupies roughly 30-40% of the frame width — NOT
foreground-dominant, NOT taking up half the frame. The room around it
reads cleanly with breathing room.
```

### E. New Single Room directive (all Shot 1)

```
SINGLE ROOM — NO DOORWAY DEPTH
This shot is set in ONE single room. NO open doorways revealing a second
room or hallway behind the product, NO see-through to another space, NO
architectural depth into another room, NO corridor visible. The wall behind
the product is a continuous solid wall.
```

Addresses Feeding Pillow second-room doorway.

### F. Negative prompt — major expansion

Added six new denials:
- `NO console tables or side tables or nightstands placed behind or directly beside the product` (Bouncer table)
- `NO oversized floral mural decals, NO large flower paintings on walls`
- `NO open doorway revealing a second room or hallway`
- `NO cloud-shaped nightlights or practical nursery nightlights as a styled element in the frame` (defensive)
- `NO deep saturated colored walls (forest green, deep teal, burnt orange, dusty rose) — walls stay in soft neutral or pale pastel tones`
- `NO color-block painted wall planes, NO studio set color planes`

## Deployment — TWO STEPS

### Step 1: Push `index_v56.html` to GitHub Pages

This activates all logic changes plus the cleaned local fallback `_A`/`_W`/`_L`. If you only do this step, the live web app pulls from Supabase — and Supabase still has the toxic entries — so the deletes from V5.6 must also run in Supabase.

### Step 2: Run `V5.6_supabase_cleanup.sql` in Supabase SQL Editor

The SQL has inspection SELECTs at the top — run those first to see your current state. Then run the 9 DELETE statements. Then the verify SELECTs at the bottom should show exactly 9 aesthetics, 9 walls, 7 lighting moods.

After both steps, the live web app and the offline batch generator are fully aligned.

## What V5.6 won't fix

- If you randomly pick `Vintage Cottagecore` it still leans rustic — should stay calmer than V5.4 era because canopies/wreaths were stripped, but tone is still "homey". Pick deliberately for nursery photos that want that vibe.
- The AI still occasionally adds small accent tables despite the `NO console tables` negative — if it persists, the next fix would be removing the `Reading Corner` zone (which mentions "low children's bookshelf" as a piece of furniture). Hold for now.
