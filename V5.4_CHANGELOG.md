# V5.4 Changelog — Removed bad aesthetics, added directives for floor/crib/pillow

## Problem identified

V5.3 mechanism filter + cleaned aesthetics fixed the worst V5.2 regressions but exposed three new issues in the 15-category hero batch:

1. **Wardrobe** — fabric pennants/garlands draped from ceiling beams. Source: Storybook `materials` includes "fabric garland".
2. **Changing Table** — coral color-block wall behind the product. Source: Playful Bright `materials` includes "rainbow wallpaper, glossy paints, distinct pops of color".
3. **Recliner Chair** — rainbow wallpaper alcove with bunting + animal decals. Same Playful Bright source.
4. **Baby Rocker** — distressed wooden dresser visible behind the product as a clear secondary hero. Source: Boutique Editorial `materials` includes "premium parquet, sculptural lighting" — AI extends to additional sculptural furniture.
5. **Pillow** — distressed weathered crib visible in shot, baby pillow scaled adult-sized. Source: same Boutique Editorial; also missing crib-quality directive and pillow-scale rule.
6. **Crib** — two-tone painted floor (charcoal + sage split). Source: no floor uniformity rule — AI freelanced on Modern Gender-Neutral.

## V5.4 changes

### 1. Aesthetics removed entirely from `_A`

- **Playful Bright (`playful`)** — produced color-block walls, rainbow wallpaper, animal-decal alcoves
- **Boutique Editorial (`boutique`)** — produced dressers as secondary hero; per Sameer's call after seeing Baby Rocker

Pool reduced from 16 → 14 aesthetics.

### 2. Aesthetic data cleaned

- **Storybook** materials — stripped "fabric garland" + added explicit anti-directive: "(no fabric garlands or pennant buntings hung from the ceiling)"

### 3. `MECHANISM_SAFE_AES` reduced

V5.3: `{scandi, coastal, japandi, warmtona, genderne, colorpop, playful, boutique}`
V5.4: `{scandi, coastal, japandi, warmtona, genderne}`

Removed `playful` and `boutique` (deleted from data); also removed `colorpop` (Scandi Color-Pop) which has saturated color-block wall as its core descriptor. The 5 remaining are all reliably clean for mechanism categories.

### 4. New directive — Floor uniformity (Shot 1, all categories)

Added to the post-CROP-SAFETY block:

```
FLOOR — UNIFORM SINGLE SURFACE
The floor is one continuous uniform surface across the entire frame — a single
material in a single tone. Acceptable: natural wood plank (oak, pine, ash),
pale stone, soft single-color rug filling the floor area. NOT ACCEPTABLE:
painted color-block floors, two-tone split flooring, contrasting painted floor
zones, multi-color painted geometry on the floor. The floor reads as a real
home floor, not a graphic-design composition.
```

This addresses the Crib two-tone floor regression and any future similar drift.

### 5. New directive — Clean modern crib (crib-anchored shots)

Added to the HERO FRAMING block when `isCribAnchored(cat)`:

```
CRIB QUALITY — CLEAN MODERN
The crib visible in the frame is a clean modern crib in painted white, light
natural oak, or pale ash. NOT distressed, NOT weathered, NOT grey-washed,
NOT antiqued, NOT reclaimed-wood texture. The crib reads as a brand-new
premium nursery crib straight from a high-end catalog.
```

Applies to: Pillow, Comforter Set, Cot Fitted Sheet, Bolster Set, Mattress, Sleeping Bag, Bassinet Bedding, Cradle Bedding Set.

### 6. New directive — Pillow scale (Pillow only)

```
PILLOW SCALE — REALISTIC BABY PROPORTIONS
The pillow is a small baby pillow approximately 30 cm wide (12 inches),
placed inside the crib. It occupies roughly 35-45% of the crib mattress
width — deliberately small relative to the mattress. The pillow is NOT
adult-pillow-sized, NOT filling the mattress width, NOT taking up half
the crib interior.
```

### 7. Version bump

V5.3 → V5.4 in title, login splash, header, comments.

## Deployment

Two steps:

1. **Push `index_v54.html` to GitHub Pages** — applies all logic changes (mechanism filter, floor rule, crib quality rule, pillow scale rule, expanded negatives, Storage in fallback `_C`)
2. **Run `V5.4_supabase_aesthetics_update.sql`** in Supabase SQL Editor — deletes Playful Bright + Boutique Editorial, updates Storybook materials text

### Batch testing (offline CSV)

No Supabase needed — `generate_v54_test_prompts.js` reads from bundled `_A` in `index_v54.html`. Already includes all V5.4 changes.

## V5.4 — what changes for the live randomization

When you click "Generate" on the live prompter with everything set to Random:

- All 14 aesthetics are equally likely for non-mechanism categories
- For mechanism categories (Bouncer, Walker, Swing, Baby Swing, Baby Rocker, Recliner Chair, Play Gym, Changing Table), only 5 aesthetics randomize: Scandinavian, Coastal, Japandi, Warm Tonal, Modern Gender-Neutral
- Manual aesthetic selection still allows any of the 14 — the filter only applies to "🎲 Random"
